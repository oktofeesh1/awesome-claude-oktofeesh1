## Content

Spec Workflow MCP Server gives AI-assisted development a structured
requirements -> design -> tasks workflow. It stores project-local spec state,
tracks approvals and revisions, records implementation logs, and can pair with a
real-time dashboard or VS Code extension for visual progress management.

The upstream README documents client setup for Claude Code, Claude Desktop,
Codex, Cursor, Continue, Windsurf, OpenCode, Cline, and Augment. The npm package
is `@pimzino/spec-workflow-mcp`, and the server accepts a target project path so
spec files can live with the project being planned.

## Features

- Sequential spec workflow for requirements, design, and implementation tasks.
- Approval process with feedback and revisions.
- Real-time web dashboard for specs, task progress, and workflow status.
- VS Code extension for integrated dashboard access.
- Implementation logs with searchable history and code statistics.
- Multi-language documentation and interface support.
- Docker dashboard deployment and sandbox-friendly `SPEC_WORKFLOW_HOME`
  configuration.
- Security controls including local binding, rate limiting, audit logging,
  security headers, CORS restrictions, and Docker hardening.

## Installation

Add the MCP server to your client with the target project path:

```json
{
  "mcpServers": {
    "spec-workflow": {
      "command": "npx",
      "args": [
        "-y",
        "@pimzino/spec-workflow-mcp@latest",
        "YOUR_PROJECT_PATH"
      ]
    }
  }
}
```

Run the dashboard separately with the upstream `--dashboard` option when visual
approval and progress tracking are needed. VS Code users can also use the
project's marketplace extension.

## Use Cases

- Ask Claude to create requirements, design, and task documents before coding.
- Track approval status and revision history for AI-generated specs.
- Use dashboard progress views to coordinate implementation work.
- Keep task implementation logs beside the project.
- Add a review checkpoint between broad product ideas and executable agent
  tasks.

## Safety and Privacy

Spec Workflow MCP can influence what an agent builds next. Treat generated
requirements, design docs, and tasks as proposals until reviewed by a person.
When the dashboard is used outside a local development machine, protect it with
TLS, authentication, and network restrictions.

Spec files, approval comments, logs, templates, code statistics, and project
paths can reveal sensitive roadmap or architecture details. Review `.spec-workflow`
contents before sharing a repository or attaching logs to public issues.
